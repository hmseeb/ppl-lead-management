# Plan 28-01: Lead Visibility

## Goal
Brokers can view a paginated list of all leads assigned to them, with delivery status details.

## Requirements
- LEAD-01: Paginated list of all leads assigned to broker
- LEAD-02: Each row shows name, vertical, credit score, funding amount, delivery status, timestamp
- LEAD-03: Broker cannot see leads assigned to other brokers (RLS + query filter)

## Approach
1. Add `fetchBrokerLeadsPaginated(brokerId, page, perPage)` to portal queries
   - Returns leads with count for pagination
   - Separately fetches latest delivery status per lead_id for the page
2. Create `leads-table.tsx` server component with table rows
3. Create `/portal/(protected)/leads/page.tsx` with pagination controls
4. Update `portal-header.tsx` to include Dashboard and Leads nav links
5. Verify with `npx tsc --noEmit`

## Delivery Status Strategy
Query deliveries table for each page of lead IDs, group by lead_id, pick best status:
- sent = success (green)
- retrying = in progress (amber)
- queued = pending (gray)
- failed = failed (red)
- failed_permanent = permanently failed (red)

Priority: sent > retrying > queued > failed > failed_permanent
