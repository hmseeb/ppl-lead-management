# Requirements: PPL Lead Management

**Defined:** 2026-03-12 (v1.0), 2026-03-13 (v1.1)
**Core Value:** Leads are matched and delivered to the right broker within seconds of arriving, every time, with full audit trail.

## v1.0 Requirements (Complete)

All 48 requirements delivered. See MILESTONES.md for details.

## v1.1 Requirements — Monitoring & Alerting

Requirements for milestone v1.1. Each maps to roadmap phases.

### Monitoring — Delivery Stats Dashboard

- [x] **MNTR-01**: Dashboard shows today's lead counts (received, assigned, unassigned)
- [x] **MNTR-02**: Dashboard shows today's delivery counts by channel (webhook, email, SMS)
- [x] **MNTR-03**: Dashboard shows today's failed delivery count with channel breakdown
- [x] **MNTR-04**: Delivery stats update in real-time via existing Supabase Realtime
- [x] **MNTR-05**: Channel health indicators show color-coded status (green/yellow/red)

### Alerts — Failure + Unassigned Notifications

- [x] **ALRT-01**: Admin GHL contact ID stored in Supabase Vault for alert delivery
- [x] **ALRT-02**: SMS alert fires when delivery hits failed_permanent, includes lead name, broker name, channel, error
- [x] **ALRT-03**: SMS alert fires when lead goes to unassigned queue, includes lead details and match failure reason
- [x] **ALRT-04**: Alert deduplication prevents duplicate SMS for same broker/reason within 15-minute window
- [x] **ALRT-05**: Reusable `send-alert` edge function serves both alert types via type discriminator

### Digest — Daily Morning Summary

- [x] **DGST-01**: Daily digest runs at 8 AM Pacific via pg_cron -> edge function
- [x] **DGST-02**: Email digest includes overnight stats (leads received, assigned, unassigned, deliveries by channel, failures)
- [x] **DGST-03**: SMS digest includes compact summary of overnight numbers
- [x] **DGST-04**: Digest delivered to admin via GHL Conversations API (email + SMS)

## Future Requirements (Deferred)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Historical delivery analytics with date range | Dashboard is for today's health, not historical analysis |
| Per-broker delivery stats on main dashboard | Already visible on broker detail pages |
| Configurable alert channels (Slack, email, push) | One admin, one channel (GHL SMS). Add if team grows |
| Alert severity levels | One event type (failed_permanent). No gradient needed |
| Alert acknowledgement / snooze / escalation | One-person operation. Manual retry endpoint is the "ack" |
| Configurable digest schedule | One admin, hardcoded 8 AM Pacific. Change cron if needed |
| Weekly/monthly roll-up digests | Defer until operational patterns emerge |
| Delivery latency metrics | Need baseline data first |
