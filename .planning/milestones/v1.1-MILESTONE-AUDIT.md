## Findings

### Actionable (non-blocking)

1. **Realtime publication not version-controlled** - The `deliveries` table subscription in RealtimeListener works because Supabase Realtime was configured during v1.0. But no migration explicitly adds the publication. Low risk (it works), but adding it to a migration would make it reproducible.

2. **Dead Vault secret** - `vault.create_secret('llsWInEk2r7jRoxhPl5T', 'admin_ghl_contact_id')` in migration 00012 creates a Vault entry, but all code reads the contact ID from the `admin_settings` table row instead. Cosmetic, zero functional impact.

3. **Traceability table stale** - REQUIREMENTS.md traceability table shows 13/14 entries as "Pending" despite all checkboxes being ticked and all verifications passing. Documentation-only issue.

4. **ROADMAP progress table stale** - Phases 6, 7, 9 show "Not started" / "0/N plans". Phase 8 is correctly marked. Documentation-only issue.
