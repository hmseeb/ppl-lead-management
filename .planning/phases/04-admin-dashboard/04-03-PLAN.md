1. Create `src/lib/queries/brokers.ts` with:
   - `fetchBrokersWithStats()`: Query all brokers, and for each broker aggregate:
     - active_orders_count: count of orders where broker_id matches and status = 'active'
     - total_leads_delivered: sum of leads_delivered across all broker's orders
     - last_delivery_date: max(last_assigned_at) across all broker's orders
     - Use a single query with Supabase select that joins orders: `brokers(*, orders(id, status, leads_delivered, last_assigned_at))`
     - Compute aggregates in TypeScript after fetch (Supabase doesn't support server-side aggregation on joins easily)
   - `fetchBrokerDetail(id: string)`: Fetch single broker with:
     - All broker fields
     - All orders for this broker (with status, leads counts, verticals, dates)
     - All leads assigned to this broker: query leads where assigned_broker_id = id, ordered by assigned_at DESC
     - For each lead, include webhook delivery status (latest delivery record)

2. Rewrite `src/components/brokers/brokers-table.tsx` (currently a basic server component table):
   - Keep as server component
   - Add new columns: Active Orders (count), Total Leads Delivered (number), Last Delivery (formatted date or 'Never')
   - Make broker Name column a Link to /brokers/[id]
   - Keep existing columns: Name, Company, Email, Phone, Status, Actions
   - Use date-fns `formatDistanceToNow` for "Last Delivery" column

3. Update `src/app/(dashboard)/brokers/page.tsx`:
   - Call fetchBrokersWithStats() instead of direct Supabase query
   - Pass stats data to BrokersTable

4. Create `src/components/brokers/broker-quick-actions.tsx` as a client component:
   - Three buttons: "Pause All Orders", "Resume All Orders", "New Order" (links to /orders/new?broker_id=X)
   - Pause All: calls a new server action `pauseAllBrokerOrders(brokerId)` that updates all active orders to paused
   - Resume All: calls `resumeAllBrokerOrders(brokerId)` that updates all paused orders to active
   - Add these two server actions to `src/lib/actions/brokers.ts` (append, don't replace existing)
   - Use sonner toast for success/error feedback
   - Disable buttons contextually (e.g. disable Pause All if no active orders)

5. Create `src/components/brokers/broker-detail.tsx` as a server component:

   - Orders section: ShadCN Table listing all broker's orders with columns: ID (short UUID), Verticals (badges), Total/Delivered/Remaining, Status (color badge), Created date. Each order ID links to /orders/[id]
   - Leads History section: ShadCN Table listing all leads received, columns: Date, Lead Name, Vertical, Credit Score, Order ID, Delivery Status (badge). Each lead name links to /leads/[id]
   - Show "No orders yet" and "No leads received yet" empty states

   - Render back link to /brokers

   - Render BrokerDetail with all data
