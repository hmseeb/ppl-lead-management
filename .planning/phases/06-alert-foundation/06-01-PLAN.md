<task type="auto">
  <name>Task 2: Create send-alert edge function with type discriminator</name>
  <files>supabase/functions/send-alert/index.ts</files>
  <action>
Create `supabase/functions/send-alert/index.ts` following the exact pattern from `deliver-ghl/index.ts`:

1. **Auth check**: Verify `Authorization: Bearer {service_role_key}` header. Return 401 if missing.

2. **Parse request body** expecting:
   ```typescript
   { type: 'delivery_failed' | 'unassigned_lead', admin_contact_id: string, ...data }
   ```

3. **Validate required fields**: `type` and `admin_contact_id` must be present. Return 400 if missing.

4. **Read GHL_API_TOKEN** from `Deno.env.get('GHL_API_TOKEN')`. Return 500 if not configured.

5. **Format SMS body based on type**:
   - `delivery_failed`: Multi-line message with "ALERT: Delivery failed permanently", lead name, broker name, channel, and error. Fields: `lead_name`, `broker_name`, `channel`, `error`.
   - `unassigned_lead`: Multi-line message with "ALERT: Lead unassigned", lead name, and reason. Fields: `lead_name`, `reason`.
   - Unknown type: Generic fallback with type name and JSON stringified data.

6. **Send SMS via GHL Conversations API** using the exact same endpoint and headers as deliver-ghl:
   ```typescript
   const GHL_BASE_URL = 'https://services.leadconnectorhq.com'
   const GHL_API_VERSION = '2021-07-28'

   fetch(`${GHL_BASE_URL}/conversations/messages`, {
     method: 'POST',
     headers: {
       'Content-Type': 'application/json',
       'Authorization': `Bearer ${ghlToken}`,
       'Version': GHL_API_VERSION,
     },
     body: JSON.stringify({
       type: 'SMS',
       contactId: admin_contact_id,
       message,
     }),
   })
   ```

7. **Handle GHL response**:
   - Success (2xx): Return `{ success: true, messageId }` with status 200.
   - Rate limited (429): Return `{ error: 'ghl_429' }` with status 429. Do NOT mark as permanent failure.
   - Other errors: Return `{ error: 'ghl_{status}', detail }` with status 502.
   - Network error: Return `{ error: 'ghl_network_error' }` with status 502.

8. **Console.error** on any GHL failure for Supabase edge function logs visibility.

Do NOT import supabase-js. This function only calls GHL. It does not read/write the database. Keep it stateless.
