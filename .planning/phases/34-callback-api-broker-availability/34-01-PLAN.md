- CREATE INDEX idx_callbacks_broker_status ON callbacks(broker_id, status) for querying pending callbacks per broker

- Apply the existing update_updated_at() trigger to callbacks table

Create `src/lib/webhooks/callback-webhook.ts`:
- Export async function fireCallbackWebhook(params: { type: 'callback_created' | 'callback_cancelled', callback: { id, scheduled_time, status, notes, created_at }, lead: { id, first_name, last_name, email, phone, vertical, credit_score, funding_amount, state }, broker: { id, first_name, last_name, email, phone, company_name, crm_webhook_url } })
- The function POSTs to broker.crm_webhook_url with JSON body containing: type (the discriminator), callback (full callback object), lead (full lead details), broker (id, name, email, phone, company)

- Fire-and-forget pattern: the API routes call this but do NOT await the result blocking the response. Use .catch(console.error) pattern from leads/incoming/route.ts
- Log errors to console.error with prefix 'callback-webhook:' but never throw (webhook failure should not block API response)
- If crm_webhook_url is null or empty, skip silently (not all brokers have CRM webhooks configured)

- Log to activity_log: event_type 'callback_created', lead_id, broker_id, details { callback_id, scheduled_time }

Create `src/app/api/callbacks/[id]/route.ts` with DELETE handler:
- Extract callback id from route params (Next.js 16 pattern: params is a Promise, await it)

- Update callback status to 'cancelled', updated_at to now()
- Fetch the associated lead and broker records for the webhook payload
- Fire webhook: call fireCallbackWebhook with type 'callback_cancelled'. Use .catch(console.error).
- Log to activity_log: event_type 'callback_cancelled', lead_id, broker_id, details { callback_id }
- Return 200 with the updated callback record
