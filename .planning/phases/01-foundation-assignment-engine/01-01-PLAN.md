    ```
    Or use the admin Supabase client to query information_schema.columns for the brokers table.

    Then create SQL migration files in `supabase/migrations/`:

    **00001_enable_extensions.sql:**
    - `CREATE EXTENSION IF NOT EXISTS pg_cron;` (may need to be enabled via Supabase dashboard SQL editor since pg_cron requires superuser)
    - `CREATE EXTENSION IF NOT EXISTS pg_net;` (same caveat)
    Note: If these fail via migration, run them manually in the Supabase SQL Editor. They require the `postgres` role.

    **00002_alter_brokers.sql:**
    - Check if `status` column exists. If it does, check its type. If it's compatible with our needs (active/paused/completed), use it. If not, add `assignment_status text NOT NULL DEFAULT 'active' CHECK (assignment_status IN ('active', 'paused', 'completed'))`.
    - Add `company text` IF NOT EXISTS.
    - The existing table already has: name, email, phone, crm_webhook_url (this IS the GHL webhook URL).
    - Use `ALTER TABLE brokers ADD COLUMN IF NOT EXISTS` for EVERY column addition. Never DROP or ALTER existing columns.

    **00003_create_tables.sql:**
    - Create `orders` table with: id (uuid PK), broker_id (FK to brokers), total_leads (int, CHECK > 0), leads_delivered (int default 0), leads_remaining (int), verticals (text[] NOT NULL), credit_score_min (int nullable), status (text NOT NULL DEFAULT 'active', CHECK IN active/paused/completed), bonus_mode (boolean default false), last_assigned_at (timestamptz nullable), created_at, updated_at. Add CHECK constraint: `CHECK (leads_remaining >= 0 OR bonus_mode = true)`.
    - Create `leads` table per research schema (id, ghl_contact_id UNIQUE, first_name, last_name, email, phone, business_name, vertical, credit_score, funding_amount, funding_purpose, state, ai_call_notes, ai_call_status, raw_payload jsonb, status text CHECK IN pending/assigned/unassigned, assigned_broker_id FK, assigned_order_id FK, assigned_at, created_at, updated_at).
    - Create `activity_log` table (id, event_type text NOT NULL, lead_id FK nullable, broker_id FK nullable, order_id FK nullable, details jsonb, created_at).
    - Create `unassigned_queue` table (id, lead_id FK NOT NULL UNIQUE, reason text NOT NULL, details text, resolved boolean default false, resolved_at timestamptz, created_at).
    - Add `updated_at` auto-update trigger function and apply to orders, leads tables.

    **00004_create_indexes.sql:**
    - All indexes from research: idx_orders_status_broker, idx_orders_active_assignment (partial), idx_leads_status, idx_leads_ghl_contact_id, idx_leads_assigned_broker, idx_activity_log_created, idx_activity_log_type, idx_unassigned_queue_resolved (partial).

    **00005_enable_rls.sql:**
    - Enable RLS on ALL new tables (orders, leads, activity_log, unassigned_queue).
    - Check if RLS is already enabled on brokers. If not, enable it BUT be careful: ppl-onboarding may use the anon key to read brokers. Add a SELECT policy for anon if enabling RLS on brokers.
    - Since this is admin-only and all mutations go through service role key, create permissive policies for the service role or use the fact that service role bypasses RLS.

    bunx supabase db push --project-ref kovcroqqudozpaocpeks

    After migrations succeed, regenerate TypeScript types:
    ```
