**In scoring.ts:** Add a new exported function `scoreLeadFull()` that returns ALL scored orders including disqualified ones. The existing `scoreLead()` function already builds the full results array internally but filters to eligible-only before returning. `scoreLeadFull()` should:

1. Use the same logic as `scoreLead()` (hard filters + scoring for eligible, zero scores for disqualified)
2. Return the COMPLETE results array (both eligible and disqualified)
3. Sort eligible orders by score DESC then fill_rate ASC (same as scoreLead), disqualified orders after eligible unsorted
4. Keep `scoreLead()` unchanged (other code may depend on its current behavior)

Implementation approach: Extract the shared scoring loop into an internal helper, have both functions use it, one filters and one doesn't. OR simpler: `scoreLeadFull()` just runs the same loop and returns all results partitioned (eligible sorted first, then disqualified).

```typescript
export function scoreLeadFull(
  lead: LeadForScoring,
  orders: OrderForScoring[]
): { eligible: ScoredOrder[]; disqualified: ScoredOrder[]; all: ScoredOrder[] } {
  // ... same loop as scoreLead building results array ...
  const eligible = results.filter(r => !r.disqualified)
    .sort((a, b) => b.score.total !== a.score.total ? b.score.total - a.score.total : a.fill_rate - b.fill_rate)
  const disqualified = results.filter(r => r.disqualified)
  return { eligible, disqualified, all: [...eligible, ...disqualified] }
}
```
