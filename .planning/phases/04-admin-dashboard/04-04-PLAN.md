  <action>
1. Create `src/lib/queries/unassigned.ts` with:
   - `fetchUnassignedQueue()`: Query unassigned_queue where resolved = false, joined with leads (all fields) via lead_id. Order by created_at DESC.
   - `fetchActiveBrokersWithOrders()`: Query brokers where assignment_status = 'active', joined with their active orders (id, verticals, leads_remaining). Used to populate the manual assignment broker/order dropdown.

2. Create `src/lib/actions/leads.ts` with server action `manualAssignLead(leadId: string, brokerId: string, orderId: string)`:
   - Validate all three IDs are non-empty UUIDs
   - Using admin client:
     a. Update the lead: set assigned_broker_id = brokerId, assigned_order_id = orderId, status = 'assigned', assigned_at = now()
     b. Decrement leads_remaining on the order (unless bonus_mode is true): fetch order first, check bonus_mode, then update
     c. Increment leads_delivered on the order
     d. Mark unassigned_queue entry as resolved: set resolved = true, resolved_at = now()
     e. Insert activity_log entry with event_type = 'manual_assignment', broker_id, order_id, lead_id, details including who/why
     f. Fire outbound webhook if broker has crm_webhook_url: Create a webhook_deliveries record (the existing pg_net trigger will fire the HTTP POST automatically)
   - revalidatePath('/unassigned') and revalidatePath('/leads')
   - Return { success: true } or { error: string }

3. Create `src/components/unassigned/unassigned-table.tsx` as a server component:
   - ShadCN Table with columns: Date (formatted), Lead Name (linked to /leads/[id]), Vertical (badge), Credit Score, Funding Amount, Reason (from unassigned_queue.reason), Details (from unassigned_queue.details, truncated), Action
   - The Reason column should display the human-readable match failure reason
   - Action column contains an "Assign" button that opens the manual assign dialog
   - Show count of unassigned leads in header
   - Empty state: "No unassigned leads. All leads have been matched." with a green checkmark icon

4. Create `src/components/unassigned/manual-assign-dialog.tsx` as a client component ('use client'):
   - Uses ShadCN Dialog component
   - Shows lead info summary at top (name, vertical, credit score)
   - Broker select dropdown: shows broker name + company, grouped by broker
   - Order select dropdown: filtered to selected broker's active orders, shows order verticals and remaining leads
   - The two dropdowns are cascading: selecting a broker filters the available orders
   - Submit button calls manualAssignLead server action
   - Show loading state during submission
   - Use sonner toast for success ("Lead assigned to [broker name]") or error
   - Close dialog on success

5. Create `src/app/(dashboard)/unassigned/page.tsx` as a server component:
   - Call fetchUnassignedQueue() and fetchActiveBrokersWithOrders()
   - Render page title "Unassigned Queue" with count
   - Render UnassignedTable passing queue data and brokers data
