Create three SQL migration files:

**00007_webhook_deliveries.sql:**
- CREATE TABLE webhook_deliveries with columns:
  - id uuid DEFAULT gen_random_uuid() PRIMARY KEY
  - lead_id uuid NOT NULL REFERENCES leads(id)
  - broker_id uuid NOT NULL REFERENCES brokers(id)
  - order_id uuid NOT NULL REFERENCES orders(id)
  - target_url text NOT NULL (the broker's crm_webhook_url at time of delivery)
  - payload jsonb NOT NULL (the lead data sent to broker, snapshot at delivery time)
  - status text NOT NULL DEFAULT 'pending' CHECK (status IN ('pending', 'sent', 'failed', 'retrying', 'failed_permanent'))
  - pg_net_request_id bigint (tracks the pg_net async request for response checking)
  - retry_count integer NOT NULL DEFAULT 0
  - error_message text
  - sent_at timestamptz
  - last_retry_at timestamptz
  - created_at timestamptz NOT NULL DEFAULT now()
  - updated_at timestamptz NOT NULL DEFAULT now()
- Apply the existing update_updated_at() trigger to webhook_deliveries
- CREATE INDEX idx_webhook_deliveries_status ON webhook_deliveries(status, retry_count) WHERE status IN ('pending', 'sent', 'retrying', 'failed')
- CREATE INDEX idx_webhook_deliveries_lead ON webhook_deliveries(lead_id)
- Enable RLS on webhook_deliveries with anon SELECT policy (matching existing pattern from 00005_enable_rls.sql) and service_role full access
- Add a comment documenting the status lifecycle: pending -> sent -> (failed -> retrying -> sent | failed_permanent)

**00008_fire_outbound_webhook.sql:**
- CREATE OR REPLACE FUNCTION fire_outbound_webhook() RETURNS trigger
- The trigger fires AFTER INSERT on webhook_deliveries for each row
- Inside the function:
  1. Skip if NEW.target_url is null or empty (broker has no webhook URL)
  2. Call net.http_post(url := NEW.target_url, body := NEW.payload, headers := '{"Content-Type": "application/json"}'::jsonb)
  3. Store the returned request_id into webhook_deliveries.pg_net_request_id
  4. UPDATE webhook_deliveries SET pg_net_request_id = request_id, status = 'sent', sent_at = now() WHERE id = NEW.id
- CREATE TRIGGER trg_fire_outbound_webhook AFTER INSERT ON webhook_deliveries FOR EACH ROW EXECUTE FUNCTION fire_outbound_webhook()
- IMPORTANT: The trigger updates the same row it was inserted on. This is safe because it's an AFTER INSERT trigger and the update targets the specific row by id.

**00009_update_assign_lead.sql:**
- CREATE OR REPLACE FUNCTION assign_lead(p_lead_id uuid) that replaces the existing function
- Copy the entire existing function from 00006 but add one new block after the activity_log insert in the "assigned" path:
  - Build the outbound payload as jsonb containing: lead_id, first_name, last_name, email, phone, business_name, vertical, credit_score, funding_amount, funding_purpose, state, ai_call_notes, ai_call_status, ghl_contact_id, assigned_at (now()), order_id, broker_id
  - Look up the broker's crm_webhook_url: SELECT crm_webhook_url INTO v_webhook_url FROM brokers WHERE id = v_order.broker_id
  - INSERT INTO webhook_deliveries (lead_id, broker_id, order_id, target_url, payload) VALUES (p_lead_id, v_order.broker_id, v_order.id, v_webhook_url, v_payload)
  - Only insert if v_webhook_url IS NOT NULL AND v_webhook_url != '' (skip delivery if broker has no webhook URL configured)
- Declare the new variables: v_webhook_url text, v_payload jsonb
- The rest of assign_lead() stays identical (advisory lock, matching, rotation, decrement, auto-complete, activity log, return)
- Add delivery_id to the return jsonb when a delivery is created
