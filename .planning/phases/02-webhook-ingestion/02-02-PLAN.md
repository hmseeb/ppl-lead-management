  <action>
Create `src/app/api/leads/update/route.ts` exporting an async `PATCH` function.

Define a Zod schema inline (or import from schemas.ts if Plan 01 already created it, but keep this self-contained):
```
const leadUpdateSchema = z.object({
  ghl_contact_id: z.string().min(1),
  ai_call_notes: z.string().optional(),
  ai_call_status: z.string().optional(),
  first_name: z.string().optional(),
  last_name: z.string().optional(),
  email: z.string().email().optional().or(z.literal('')),
  phone: z.string().optional(),
  business_name: z.string().optional(),
  vertical: z.string().optional(),
  credit_score: z.coerce.number().int().min(300).max(850).optional(),
  funding_amount: z.coerce.number().positive().optional(),
  funding_purpose: z.string().optional(),
  state: z.string().optional(),
})
```

The handler should:

1. Parse JSON body. If invalid JSON, return 400 `{ error: 'invalid_json' }`.

2. Validate with leadUpdateSchema.safeParse. If invalid, return 400 with error details.

3. Extract `ghl_contact_id` from validated data. Use it to find the existing lead.

4. Query: `supabase.from('leads').select('id').eq('ghl_contact_id', ghl_contact_id).maybeSingle()`. If no lead found, return 404 `{ error: 'lead_not_found', ghl_contact_id }`.

5. Build update object from validated data. Strip `ghl_contact_id` (not updatable). Strip any undefined/null values so we only update fields that were actually provided. CRITICAL: Do NOT include `assigned_broker_id`, `assigned_order_id`, `assigned_at`, or `status` in the update. These are owned by the assignment engine.

6. Update lead: `supabase.from('leads').update(updateFields).eq('id', lead.id)`. Also store `raw_payload: body` to capture the update payload.

7. Return 200 `{ lead_id: lead.id, updated_fields: Object.keys(updateFields) }`.

Use `createAdminClient()` (service role) since this is an external webhook endpoint.
