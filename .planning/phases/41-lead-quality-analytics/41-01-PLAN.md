Export type `CreditTierDistribution`:
```ts
export type CreditTierDistribution = {
  tiers: { tier: string; min: number; max: number | null; count: number; color: string }[]
  total: number
}
```

The tiers are fixed:
- `{ tier: '500-599', min: 500, max: 599, color: '#f43f5e' }` (rose)
- `{ tier: '600-679', min: 600, max: 679, color: '#f59e0b' }` (amber)
- `{ tier: '680+', min: 680, max: null, color: '#10b981' }` (emerald)

Query: Select `credit_score` from leads table where `assigned_broker_id = brokerId`, `credit_score IS NOT NULL`, and date-range filter on `assigned_at` using `getPortalDateRange`. Count leads falling into each tier bucket in TypeScript (same approach as fetchBrokerLeadVolumeTrend). Include leads with credit_score < 500 in an "Under 500" tier only if any exist (tier: 'Under 500', min: 0, max: 499, color: '#6b7280'). Always return all 3 main tiers even if count is 0.

2. **fetchBrokerVerticalMix(brokerId, dateFilters?)**

Export type `VerticalMixData`:
```ts
export type VerticalMixData = {
  verticals: { vertical: string; count: number; percent: number }[]
  total: number
}
```

Query: Select `vertical` from leads where `assigned_broker_id = brokerId`, `vertical IS NOT NULL`, date-range filter on `assigned_at`. Group by vertical in TypeScript, compute count and percentage. Sort descending by count. Return total.

Both functions follow the same pattern as existing queries: `createAdminClient()`, `getPortalDateRange(dateFilters ?? {})`, null-safe returns.
