Refactor the existing RealtimeListener component to debounce `router.refresh()` calls. The current implementation calls `router.refresh()` directly on every postgres_changes event with no throttling.

**Changes:**

1. Add `useRef` and `useCallback` to the React imports (alongside existing `useEffect`).
2. Create two refs:
   - `timeoutRef = useRef<NodeJS.Timeout | null>(null)` for the debounce timer
   - `lastRefreshRef = useRef<number>(0)` for tracking last refresh timestamp
3. Create a `debouncedRefresh` callback wrapped in `useCallback` with `[router]` dependency:
   - Clear any existing timeout via `clearTimeout(timeoutRef.current)`
   - Check elapsed time since last refresh: `Date.now() - lastRefreshRef.current`
   - If elapsed > 2000ms (max wait), refresh immediately: set lastRefreshRef to now, call `router.refresh()`, return
   - Otherwise, set a 500ms timeout that updates lastRefreshRef and calls `router.refresh()`
4. Replace all 5 `() => router.refresh()` callbacks with `debouncedRefresh` in the channel subscriptions (leads, orders, activity_log, deliveries, unassigned_queue).
5. In the cleanup function (useEffect return), add `if (timeoutRef.current) clearTimeout(timeoutRef.current)` before `supabase.removeChannel(channel)`.
6. Update the `useEffect` dependency array to `[debouncedRefresh]` instead of `[router]` (since debouncedRefresh already depends on router via useCallback).

**Behavior:**
- First event in a quiet period: refreshes within 2s (immediate if > 2s since last refresh)
- Rapid successive events within 500ms: only the last one triggers refresh
- Maximum delay between event and refresh: 2 seconds (the max wait cap)

Do NOT change the channel subscriptions themselves (same tables, same event filters). Do NOT change the component name or export. The component still returns null.
