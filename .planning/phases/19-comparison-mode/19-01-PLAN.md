```ts
export function getPreviousDateRange(filters: DashboardFilters): { from: string; to: string }
```

Logic: Take the current date range from `getDateRange(filters)`, calculate its duration in milliseconds, then shift both `from` and `to` backward by that duration. This handles all cases uniformly:
- "today" (1 day) compares to yesterday
- "7d" compares to prior 7 days
- "30d" compares to prior 30 days
- "90d" compares to prior 90 days
- Custom range compares to an equal-length range immediately preceding it

Use `differenceInMilliseconds` from date-fns for the shift calculation. Return ISO strings like `getDateRange` does.

Also add `compare?: string` to the `DashboardFilters` interface (string because nuqs parseAsString, truthy = enabled).

```ts
const METRIC_DIRECTION: Record<string, 'positive' | 'negative'> = {
```

**3. Add a `dataKey` field to CardConfig** that maps each card to its KpiData field name. The existing `value` already comes from `data.leadsToday` etc, but we need the key string to look up previousData. Add to each card config:
- Leads: `dataKey: 'leadsToday'`
- Assigned: `dataKey: 'assignedCount'`

- Failed Deliveries: `dataKey: 'failedDeliveries'`

- Active Orders: `dataKey: 'activeOrders'`

**4. Create DeltaBadge component** (inline in kpi-cards.tsx, not a separate file):

```ts
function DeltaBadge({ current, previous, direction }: {
  current: number
  previous: number
  direction: 'positive' | 'negative'
})
```

Logic:
- If previous is 0 and current is 0: return null (no badge)
- Calculate delta: `current - previous`
- Calculate percentage: previous > 0 ? `Math.round((delta / previous) * 100)` : (current > 0 ? 100 : 0)
- Determine color:
  - If delta === 0: neutral (gray/muted)
  - If direction === 'positive': delta > 0 = green, delta < 0 = red
  - If direction === 'negative': delta > 0 = red, delta < 0 = green (inverted!)
- Render: A small inline badge below or beside the value. Show arrow (up/down) + percentage. Use Tailwind classes:
  - Green: `text-emerald-500` with up arrow
  - Red: `text-red-500` with down arrow
  - Neutral: `text-muted-foreground` with dash
- Format: Show as `+12%` or `-8%` with a small triangle arrow character (use Unicode or lucide TrendingUp/TrendingDown icons sized at 12px).
- Badge should be compact: `text-[10px] font-medium` to fit under the large number without crowding.

**5. Render DeltaBadge in each card:**

Inside the card's CardContent, after the main value div, conditionally render:

```tsx
{previousData && (
  <DeltaBadge
    current={card.value}
    previous={previousData[card.dataKey as keyof KpiData] as number}
    direction={METRIC_DIRECTION[card.dataKey] ?? 'positive'}
  />
)}
```

Place it right after the value `<div>` and before the subtitle `<p>`. This puts the delta badge between the big number and the description text.

**6. For Rejected Rate specifically:** The `value` is `data.rejectedRate` (a percentage), and `previousData.rejectedRate` is also a percentage. The delta should show percentage POINT change, not percentage of percentage. So for rejected rate, show `+5pp` or `-3pp` (percentage points). Handle this by checking if `dataKey === 'rejectedRate'` and using absolute difference instead of relative percentage.
