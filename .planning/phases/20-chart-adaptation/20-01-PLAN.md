  <action>
**Current state:** `fetchLeadVolume` already accepts filters and uses `getDateRange` to compute from/to. It loops through each day in the range and makes a separate count query per day. This works fine for 7d but creates 90 queries for 90d and produces an unreadable chart.

**Changes needed:**

1. After computing `totalDays`, determine the bucketing strategy:
   - If `totalDays <= 30`: use **daily** buckets (current behavior)
   - If `totalDays > 30`: use **weekly** buckets (group into 7-day chunks)

2. For weekly bucketing:
   - Calculate the number of weeks: `Math.ceil(totalDays / 7)`
   - Loop through weeks instead of days
   - Each bucket spans 7 days (except possibly the last one which may be shorter)
   - Label format: `"MMM d"` showing the start date of each week bucket
   - Query counts for each week bucket using `gte(weekStart)` and `lt(weekEnd)`

3. Return type stays the same: `{ date: string; label: string; count: number }[]`

4. Also return metadata about the bucketing: add a `bucketType` field to the return so the chart knows whether it's showing daily or weekly data. Change return type to `{ data: { date: string; label: string; count: number }[]; bucketType: 'daily' | 'weekly' }`.

Actually, simpler approach: keep the return as a flat array but also return the bucket type alongside. Or just let the chart figure it out from data length. Cleanest: return an object `{ data: ChartData[]; bucketType: 'daily' | 'weekly'; totalDays: number }`.
