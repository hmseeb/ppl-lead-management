<feature>
  <name>Lead Scoring Engine</name>
  <files>src/lib/assignment/scoring.ts, src/lib/assignment/scoring.test.ts</files>
  <behavior>
    The `scoreLead()` function accepts a lead object and an array of order objects (with joined broker data).
    It returns a sorted array of ScoredOrder results, or null if no eligible order exists.

    **Input types:**
    ```typescript
    interface LeadForScoring {
      credit_score: number | null
      funding_amount: number | null
      vertical: string | null
    }

    interface OrderForScoring {
      id: string
      broker_id: string
      status: string
      verticals: string[]
      credit_score_min: number | null
      leads_remaining: number
      leads_delivered: number
      total_leads: number
      bonus_mode: boolean
      loan_min: number | null
      loan_max: number | null
      priority: string  // 'high' | 'normal'
      broker_assignment_status: string
    }
    ```

    **Hard filters (order is EXCLUDED if any fail):**
    1. TIER-01: credit_score_min >= 680 AND lead.credit_score < 680 -> excluded
    2. TIER-02: credit_score_min >= 600 AND lead.credit_score < 600 -> excluded (Note: 600-min orders accept 600+ including 680+)
    3. ORDER-05: lead.funding_amount outside [loan_min, loan_max] -> excluded (null bounds = no limit on that side)
    4. Existing filters from SQL: order status = 'active', broker assignment_status = 'active', vertical match, capacity (leads_remaining > 0 OR bonus_mode)

    **Scoring components (SCORE-01 through SCORE-08):**
    - Credit Fit (max 40pts): (lead.credit_score - order.credit_score_min) / (850 - order.credit_score_min) * 40 (SCORE-02)
      - If credit_score_min is null, award full 40pts
    - Capacity (max 30pts): (1 - fill_rate) * 30 where fill_rate = leads_delivered / total_leads (SCORE-03)
    - Tier Match (max 20pts): 20pts exact match (680+ lead to 680-min, 600-679 lead to 600-min), 10pts fallback (680+ lead to 600-min) (SCORE-04)
    - Loan Fit (max 10pts): 10pts when funding_amount within [loan_min, loan_max] (SCORE-05)
      - If both loan_min and loan_max are null, award full 10pts
    - Priority Bonus: +8pts for priority = 'high' (SCORE-06)
    - Urgency Bonus: +5pts when fill_rate > 0.8, -5pts when fill_rate < 0.1 (SCORE-07)

    **Tie-breaker (SCORE-08):** Lowest fill_rate wins (most capacity remaining)

    **Test cases (input -> expected):**
    - lead(credit=720) + [order(min=680), order(min=600)] -> order(min=680) wins (20pts vs 10pts tier match)
    - lead(credit=650) + [order(min=680), order(min=600)] -> order(min=680) excluded, order(min=600) selected
    - lead + [order(fill=0.2), order(fill=0.8)] -> order(fill=0.2) wins on capacity (24pts vs 6pts)
    - lead + [order(priority=high), order(priority=normal)] -> high wins by +8pts
    - lead(funding=50000) + [order(loan_min=10000, loan_max=100000), order(loan_min=200000, loan_max=500000)] -> second order excluded
    - lead + [order(fill=0.85), order(fill=0.5)] -> first gets +5 urgency bonus
    - lead + [order(fill=0.05), order(fill=0.5)] -> first gets -5 urgency penalty
    - Two orders with identical scores but different fill_rates -> lower fill_rate wins (tie-breaker)
  </behavior>
  <implementation>

    5. Export ScoreBreakdown type (credit_fit, capacity, tier_match, loan_fit, priority_bonus, urgency_bonus, total) for Phase 16 audit trail

    Use bun test for the test runner. Tests go in scoring.test.ts using describe/it/expect.
  </implementation>
