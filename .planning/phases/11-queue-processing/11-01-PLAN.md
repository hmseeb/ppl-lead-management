**Section 1: process_queued_deliveries() function**

CREATE OR REPLACE FUNCTION process_queued_deliveries() that:

1. Declares variables: v_delivery record, v_request_id bigint, v_supabase_url text, v_service_key text, v_released int default 0
2. Fetches vault secrets upfront (supabase_url, service_role_key) for GHL channel deliveries
3. Loops over queued deliveries:
   ```sql
   FOR v_delivery IN
     SELECT d.*
     FROM deliveries d
     WHERE d.status = 'queued'
     ORDER BY d.created_at ASC
   LOOP
   ```
4. For each delivery, checks `is_within_contact_hours(v_delivery.broker_id)`. If FALSE, CONTINUE (skip, broker still out of window).
5. If TRUE (broker now in-window), fires the delivery based on channel:

   - **crm_webhook channel**: Call net.http_post directly with v_delivery.target_url and v_delivery.payload (same as fire_outbound_webhook). Then UPDATE the delivery row: status='sent', sent_at=now(), pg_net_request_id=v_request_id.
   - **email/sms channels**: Call net.http_post to the edge function URL (v_supabase_url || '/functions/v1/deliver-ghl') with the same payload structure as fire_ghl_delivery (delivery_id, channel, ghl_contact_id, payload). Then UPDATE status='pending' (edge function handles final status).
   - Skip delivery if crm_webhook but target_url is null/empty.
   - Skip GHL channels if vault secrets are null (same guard as fire_ghl_delivery).

6. Increment v_released counter for each fired delivery.
7. After loop, RETURN v_released (integer return type for logging).

IMPORTANT design notes:
- Existing triggers are AFTER INSERT only, so changing status from 'queued' to 'pending' via UPDATE will NOT re-trigger them. The function must fire deliveries directly (same pattern as process_webhook_retries in migration 00011).
- FIFO order is enforced by ORDER BY created_at ASC in the query.
- is_within_contact_hours() already handles anytime/business_hours/custom/weekend_pause/timezone logic. Just call it.
- Add a COMMENT ON FUNCTION explaining its purpose.

**Section 2: pg_cron schedule**

```sql
SELECT cron.schedule(
  'process-queued-deliveries',
  '*/5 * * * *',
  'SELECT process_queued_deliveries()'
);
```

Follow the exact cron.schedule pattern used in migrations 00010 and 00013 (simple string SQL body for a function that returns a scalar).
