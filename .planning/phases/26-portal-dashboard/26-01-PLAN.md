# Plan 26-01: Portal Dashboard

## Goal
Brokers see active orders, recent leads, spending, and delivery health on their portal home.

## Requirements
- DASH-01: Active orders with progress bars (leads_delivered / total_leads)
- DASH-02: Last 20 leads with name, vertical, credit score, timestamp
- DASH-03: Spend summary (all-time, this month, active order value)
- DASH-04: Delivery health (webhook/email/SMS success rates)

## Approach
1. Add 4 query functions to `src/lib/portal/queries.ts`
2. Create `src/components/portal/dashboard-cards.tsx` with 4 card components
3. Replace portal home placeholder with real dashboard grid
4. Verify with `npx tsc --noEmit`

## Spend Calculation Strategy
Since orders don't have `total_price_cents`, we calculate spend by:
- Counting delivered leads per vertical/credit_tier
- Looking up prices from `lead_prices` table
- Summing for all-time and this-month totals
- Active order value = sum of (total_leads * estimated price per lead) for active orders

Fallback: if no prices configured, show $0.
