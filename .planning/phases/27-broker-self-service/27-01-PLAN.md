# Plan 27-01: Broker Self-Service

## Goal
Brokers can manage their active orders (pause/resume) and delivery preferences (webhook URL, delivery methods, contact hours, timezone) without contacting admin.

## Requirements
- SELF-01: Broker can pause an active order and resume it, status updates immediately
- SELF-02: Broker can update webhook URL and delivery method preferences (webhook/email/SMS)
- SELF-03: Broker can update contact hours and timezone from settings page
- SELF-04: Broker cannot cancel or delete a paid order (controls absent from UI)

## Approach

### 1. Server Actions (`src/lib/actions/portal-self-service.ts`)
- `brokerPauseOrder(orderId)` - validates session + ownership, sets status to "paused"
- `brokerResumeOrder(orderId)` - validates session + ownership, sets status to "active"
- `updateBrokerSettings(data)` - validates session, updates delivery_methods, crm_webhook_url, delivery_email, delivery_phone, contact_hours, custom_hours_start, custom_hours_end, weekend_pause, timezone
- All actions use `requireBrokerSession()` and `assertBrokerOwnsOrder()` guards

### 2. Portal Query (`src/lib/portal/queries.ts`)
- `getPortalBrokerSettings(brokerId)` - returns delivery + contact hour fields

### 3. Orders Page (`/portal/orders`)
- Server component fetches all broker orders via `getPortalOrders()`
- Client component renders table with status badges and pause/resume buttons
- No cancel or delete buttons (SELF-04 requirement)

### 4. Settings Page (`/portal/settings`)
- Server component fetches broker settings via `getPortalBrokerSettings()`
- Client form with delivery methods checkboxes, webhook URL input, contact hours select, timezone select
- Submit calls `updateBrokerSettings()` server action

### 5. Portal Navigation
- Add Dashboard, Orders, Settings links to portal header
- Active state based on current pathname
