<objective>
Create the assign_lead() Postgres function, the atomic heart of the entire system. This function uses advisory locks to serialize lead assignment, matches leads to eligible broker orders using weighted round-robin rotation, decrements counters, auto-completes depleted orders, logs all decisions, and queues unmatched leads with detailed failure reasons.

Purpose: This is the most critical code in the entire project. If this function is wrong, leads go to the wrong brokers, counters drift, and the business breaks. It must be correct, atomic, and tested before any webhook code calls it.

Output: Working assign_lead() Postgres function deployed to Supabase, tested with seed data and concurrent calls.

    **00006_assign_lead_function.sql:**

    Part A: `build_match_failure_reason(p_lead record)` RETURNS text

    Part B: `assign_lead(p_lead_id uuid)` RETURNS jsonb

    The assign_lead function must:

    1. `PERFORM pg_advisory_xact_lock(1, 0);`
       Use the two-integer form (namespace=1, id=0) to avoid collision with GoTrue's single-bigint locks. This is a GLOBAL lock, serializing ALL assignments. This is correct because we need globally consistent rotation ordering.

    2. Fetch the lead: `SELECT * INTO v_lead FROM leads WHERE id = p_lead_id;`
       If not found, return `{'status': 'error', 'reason': 'lead_not_found'}`.

    3. Find the best matching active order using weighted rotation:
       ```sql
       SELECT o.* INTO v_order
       FROM orders o
       JOIN brokers b ON b.id = o.broker_id
       WHERE o.status = 'active'
         AND b.status = 'active'  -- (or b.assignment_status, depending on Plan 01 column choice)
         AND (v_lead.vertical = ANY(o.verticals) OR 'All' = ANY(o.verticals))
         AND (o.credit_score_min IS NULL OR v_lead.credit_score >= o.credit_score_min)
         AND (o.leads_remaining > 0 OR o.bonus_mode = true)
       ORDER BY
         (o.leads_remaining::float / GREATEST(o.total_leads, 1)) DESC,
         o.last_assigned_at ASC NULLS FIRST
       LIMIT 1;
       ```

       CRITICAL: The `'All' = ANY(o.verticals)` handles orders that accept all verticals.
       CRITICAL: `credit_score_min IS NULL` handles orders with no credit score requirement.
       CRITICAL: `bonus_mode = true` allows orders with 0 remaining to keep receiving if bonus is on.
       CRITICAL: The ORDER BY formula: higher remaining/total ratio = higher priority, tiebroken by least recently assigned. This IS weighted round-robin.

    4. If no matching order found:
       - Call build_match_failure_reason(v_lead) to get detailed reason
       - Update lead status to 'unassigned'
       - Insert into unassigned_queue with lead_id, reason, and detailed text
       - Insert into activity_log with event_type 'lead_unassigned'
       - Return `{'status': 'unassigned', 'reason': v_reason}`

    5. If matching order found, assign the lead:
       - UPDATE leads SET assigned_broker_id, assigned_order_id, assigned_at = now(), status = 'assigned'
       - UPDATE orders SET leads_delivered = leads_delivered + 1, leads_remaining = CASE WHEN bonus_mode THEN leads_remaining ELSE leads_remaining - 1 END, last_assigned_at = now()
       - Auto-complete check: UPDATE orders SET status = 'completed' WHERE id = v_order.id AND leads_remaining <= 0 AND bonus_mode = false (this is safe because we just decremented)
       - Insert into activity_log with event_type 'lead_assigned', including broker_id, order_id, and details (vertical, credit_score, remaining count, bonus_mode)
       - Return `{'status': 'assigned', 'broker_id': v_order.broker_id, 'order_id': v_order.id}`

       export interface AssignmentResult {
         status: 'assigned' | 'unassigned' | 'error'
         broker_id?: string
         order_id?: string
         reason?: string
       }

       export async function assignLead(leadId: string): Promise<AssignmentResult> {
         const supabase = createAdminClient()
         const { data, error } = await supabase.rpc('assign_lead', {
           p_lead_id: leadId,

         if (error) throw new Error(`Assignment failed: ${error.message}`)
