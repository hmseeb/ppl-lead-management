**1. Create `src/lib/types/portal-filters.ts`:**

Define a portal-specific filter interface (simpler than admin -- no broker_id, vertical, compare fields since portal is always single-broker-scoped):

```ts
export interface PortalDateFilters {
  date_preset?: string
  date_from?: string
  date_to?: string
}
```

Export `PORTAL_DATE_PRESETS` array with same values as admin: `[{ value: 'today', label: 'Today' }, { value: '7d', label: '7d' }, { value: '30d', label: '30d' }, { value: '90d', label: '90d' }]`. Use shorter labels for pill style (e.g., "7d" not "7 Days").

Export `getPortalDateRange(filters: PortalDateFilters): { from: string; to: string }` -- reuse the same logic as admin's getDateRange from date-fns (startOfDay, subDays). Import from date-fns directly, do NOT import from dashboard-filters.ts to keep portal types independent. Default preset is '30d' (not 'today' like admin -- brokers care about trends more than today's snapshot).

**2. Create `src/components/portal/portal-date-filters.tsx`:**

A `'use client'` component that renders a polished, client-facing date filter bar. This is NOT a copy of the admin filter bar. Design requirements:

- Pill-style preset buttons rendered as a segmented control with rounded-full corners, subtle background, and smooth transitions. Use `bg-primary text-primary-foreground` for the active pill, `text-muted-foreground hover:text-foreground` for inactive.
- Custom date range: two native date inputs (type="date") styled with ShadCN Input component, appearing after the pill group with a subtle "Custom" label or divider.
- When a custom date is set, deselect all pill presets. When a pill is selected, clear custom dates.
- Use nuqs `useQueryState` with `parseAsString` for `date_preset`, `date_from`, `date_to` URL params. Use `{ shallow: false }` for server-side revalidation (same pattern as admin dashboard-filters.tsx and leads-filters.tsx).
- Include a clear/reset button (X icon) that only shows when non-default filters are active.
- Layout: horizontal flex, responsive wrapping. Compact height. The component should feel like a premium SaaS product, not an admin tool.
- Import and use PORTAL_DATE_PRESETS from portal-filters.ts.
- Do NOT include broker dropdown, vertical filter, or compare toggle (those are admin-only concerns).

**1. Update `src/lib/portal/queries.ts`:**

Add date range filtering to the following existing queries. Import `getPortalDateRange` and `PortalDateFilters` from `@/lib/types/portal-filters`.

For each query that should be date-filterable, add an optional `dateFilters?: PortalDateFilters` parameter. When present, call `getPortalDateRange(dateFilters)` to get `{ from, to }` and add `.gte('assigned_at', from).lte('assigned_at', to)` (or the appropriate date column) to the Supabase query.

Queries to update:
- `fetchBrokerRecentLeads(brokerId, limit, dateFilters?)` -- filter on `assigned_at`. This makes the recent leads card date-aware.
- `fetchBrokerSpendSummary(brokerId, dateFilters?)` -- when dateFilters provided, calculate spend within that date range instead of the hardcoded "all time" / "this month" split. Add a `totalInRangeCents` field to SpendSummary type.
- `fetchBrokerDeliveryHealth(brokerId, dateFilters?)` -- filter on `created_at` for deliveries.
- `fetchBrokerMonthlySpend(brokerId, months?, dateFilters?)` -- when dateFilters provided, use the date range instead of "last N months" cutoff.

Do NOT modify `fetchBrokerActiveOrders` (active orders are not date-filtered, they show current state).
Do NOT modify `getPortalBroker`, `getPortalOrders`, `getPortalBrokerSettings`, `fetchLeadDeliveryAttempts`, `fetchBrokerLeadsPaginated`, `fetchBrokerBillingOrders` -- these are not affected by the dashboard date filter.

Keep all existing function signatures backward-compatible. The new dateFilters param is always optional. When omitted, behavior is unchanged.

**2. Update `src/app/portal/(protected)/page.tsx`:**

- Add `export const dynamic = 'force-dynamic'` at top (same as admin dashboard).
- Accept `searchParams` prop (same pattern as admin dashboard page): `searchParams: Promise<Record<string, string | undefined>>`.
- Parse searchParams into a `PortalDateFilters` object.
- Wrap the page content in `<NuqsAdapter>` (import from `nuqs/adapters/next/app`).

- Pass the parsed dateFilters to `fetchBrokerRecentLeads`, `fetchBrokerSpendSummary`, `fetchBrokerDeliveryHealth`, and `fetchBrokerMonthlySpend`.
- Do NOT change the layout structure of the dashboard cards. Only add the filter bar and wire the date params through.
