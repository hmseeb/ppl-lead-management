# Plan 29-01: Billing + Revenue

## Overview

Three requirements:
1. **BILL-01**: Broker billing page with order history (amount, date, status, leads delivered)
2. **BILL-02**: Stripe receipt links per paid order
3. **BILL-03**: Admin revenue summary (total, by broker, by vertical)

## Tasks

### Task 1: Portal billing query + receipt API
- Add `fetchBrokerBillingOrders(brokerId)` to `src/lib/portal/queries.ts`
  - Select all orders with stripe/pricing columns, all statuses (not just active)
  - Returns id, verticals, total_leads, leads_delivered, status, total_price_cents, price_per_lead_cents, stripe_checkout_session_id, created_at
- Add server action `getReceiptUrl(orderId)` in `src/lib/actions/billing.ts`
  - Validates broker session owns the order
  - Retrieves stripe checkout session -> payment_intent -> latest_charge -> receipt_url
  - Returns the receipt URL for client-side redirect

### Task 2: Billing page + component
- Create `src/app/portal/(protected)/billing/page.tsx` (server component)
- Create `src/components/portal/billing-table.tsx` (client component for receipt button)
- Table columns: Date, Vertical, Leads (delivered/total), Amount, Status, Receipt
- Receipt column: button that calls getReceiptUrl server action and opens in new tab
- Empty state when no orders exist

### Task 3: Portal navigation update
- Add "Billing" link to portal header navigation
- Route: `/portal/billing`

### Task 4: Admin revenue summary
- Add `fetchRevenueSummary()` to `src/lib/queries/dashboard.ts`
  - Total revenue: sum of total_price_cents for orders with status != 'pending_payment'
  - Revenue by broker: group by broker_id, join brokers for names
  - Revenue by vertical: group by verticals[0]
- Create `src/components/admin/revenue-summary.tsx`
  - Three sections: total, by broker table, by vertical table
- Add to admin dashboard page below existing content

## Key Decisions
- Receipt URLs fetched on-demand from Stripe API (not stored). Avoids migration, keeps data fresh
- Server action for receipt URL keeps Stripe secret key server-side only
- Revenue summary uses total_price_cents from orders table (already stored at checkout)
- Orders with status='pending_payment' excluded from revenue (payment not completed)
- Admin revenue component is a server component, no client JS needed
